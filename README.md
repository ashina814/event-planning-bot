# Event Planning Bot

Discord「イベント企画室」フォーラム向けの運営補助 bot です。

このリポジトリは設計書 v0.1 の Phase 1-3 MVP を実装対象にしています。

## 実装済み MVP

- `/event new <title> [series]`
  - フォーラムに `【企画中】` prefix 付きのイベントスレッドを作成
  - 親投稿テンプレを投稿
  - bot のコントロールパネル embed を常駐投稿
  - `events` / `series` を SQLite に保存
- `/events`
  - 進行中イベント一覧を ephemeral 表示
- `/help`
  - 簡易ヘルプ
- コントロールパネル
  - 担当変更
  - 状態変更
  - 開催日時設定
  - 引き継ぎ宣言
- 告知文
  - 下書き作成
  - 履歴一覧とプレビュー
  - 公式告知チャンネルへの plain text 即時転送
  - JST 日時指定の予約投稿
  - `scheduled_jobs` の `announcement_post` handler
- タイムキーパー
  - コントロールパネルからセットアップ
  - `HH:mm セクション名` のタイムテーブル入力
  - 事前通知と開始通知を `scheduled_jobs` に登録
  - 当日操作用の [開始] / [次へ] ボタン
  - 終了時にシリーズイベントなら `section_history` に実測時間を保存
- 参加者カウント
  - リアクション方式
  - 投稿方式
  - カウントキャッシュ更新
  - 手動再集計
  - reaction add/remove と message create/delete の listener
- ToDo
  - イベント単位の ToDo 一覧
  - ToDo 追加
  - 完了 / 未完了切替
  - 作成者または統括による削除
  - 期限当日のリマインダー
  - 議事録チャンネルからの ToDo 候補検出
  - 議事録 ToDo 候補の採用 / 破棄
- 出費
  - イベント単位の出費一覧と合計
  - 出費 / 補填・返金の記録
  - 証明画像フォローアップ
  - `CH_EXPENSE_LOG` への整形投稿
  - 単発 / イベント / 月次の閾値アラート
- カレンダー・統計・振り返り
  - `/events` から月別カレンダー表示
  - `/events` から月別統計表示
  - 開催日時 +24時間の振り返りリマインダー
- シリーズ
  - `/event new` の `[series]` で自動作成
  - `/events` で登録済みシリーズを確認
  - セクション定義と履歴保存用 repo を実装
- SQLite スキーマ
  - 設計書の v0.1 DDL を `src/db/schema.sql` に反映
- 汎用スケジューラ
  - `scheduled_jobs` から due job を 1 秒 tick で処理
  - 現時点の実装 handler は `event_auto_progress`
- JST 時刻ヘルパー
  - 曜日テストケース込み

## セットアップ

Node.js は設計書どおり **22 LTS** を使ってください。`better-sqlite3@11` は Node 24 だと prebuild がなく、環境によってはソースビルドに落ちます。

```bash
pnpm install
cp .env.example .env
```

`.env` に Discord Bot Token、Guild ID、チャンネル ID、ロール ID を設定してください。

必須:

- `DISCORD_TOKEN`
- `CLIENT_ID`
- `GUILD_ID`
- `CH_EVENT_FORUM`
- `CH_EVENT_ANNOUNCE`
- `ROLE_EVENT_LEAD`
- `ROLE_EVENTER`

## Discord 側で必要な権限

bot には少なくとも以下が必要です。

- アプリケーションコマンドの使用
- フォーラムチャンネルへの投稿
- スレッド作成
- スレッド内メッセージ送信
- メッセージ編集
- スレッド名変更
- メンバー取得
- 公式告知チャンネルへの送信

`Message Content Intent` は議事録 ToDo 検出に使います。Discord Developer Portal で有効化してください。

## 告知文の使い方

イベントスレッドのコントロールパネルから「📢 告知文」を押します。

下書き作成は主担当・告知担当・イベント統括が可能です。公式告知チャンネルへの即時転送と予約投稿は、設計書どおりイベント統括のみ可能です。

予約日時は JST で `YYYY-MM-DD HH:mm` 形式です。例: `2026-07-01 22:00`

告知本文は embed にせず、そのまま plain text で投稿します。

## 開催日時の設定

イベントスレッドのコントロールパネルから「📅 日時」を押します。

日時は JST で `YYYY-MM-DD HH:mm` 形式です。例: `2026-07-01 22:00`

開催日時を設定すると、カレンダー、参加者カウント締切、タイムキーパー基準日、振り返りリマインダーに使われます。イベントが告知済みの場合は、開催時刻で `in_progress` へ自動遷移する job も登録されます。

## タイムキーパーの使い方

イベントスレッドのコントロールパネルから「⏱️ タイマー」を押します。

タイマー操作は主担当・司会・イベント統括が可能です。

セットアップでは通知先チャンネル、メンション対象ロール、事前通知分、タイムテーブルを入力します。通知先を空にするとイベントスレッドに通知します。メンション対象を空にすると主担当にメンションします。

タイムテーブル形式:

```text
22:00 集合
22:05 告知
22:15 自己紹介
22:48 ルール説明
22:50 手上げて
```

イベント開催日時が設定済みならその日付、未設定ならセットアップ当日の JST として解釈します。

## 参加者カウントの使い方

イベントスレッドのコントロールパネルから「👤 参加者」を押します。

設定は主担当・イベント統括が可能です。

リアクション方式:

```text
方式: reaction
対象: DiscordメッセージURL または メッセージID
絵文字設定: ⭕:参加,✨:興味,❌:不参加
締切 JST: 空なら開催日時
```

投稿方式:

```text
方式: post
対象: チャンネルID または スレッドID
絵文字設定: 空
締切 JST: 空なら開催日時
```

投稿方式は対象チャンネル/スレッドの直近 100 件を再集計します。リアクション方式の再集計は現在のリアクション数を通常参加として読み直し、設定後の追加/削除イベントで遅刻分を更新します。

## ToDo の使い方

イベントスレッドのコントロールパネルから「✅ ToDo」を押します。

ToDo 追加はイベンターまたはイベント担当者が可能です。完了/未完了の切替はイベンター、作成者、担当者が可能です。削除は作成者またはイベント統括のみ可能です。

期限は `YYYY-MM-DD` または `YYYY-MM-DD HH:mm` 形式です。日付だけの場合は JST 9:00 にリマインダーを送ります。

議事録 ToDo 検出:

- `CH_MINUTES` の新規投稿を監視します。
- `【ToDo】` または `【To Do】` セクションを探します。
- `・` または `- [ ]` で始まる行を候補として抽出します。
- 候補は未紐付け ToDo として保存し、`CH_INTERNAL_ANNOUNCE` にイベント統括メンション付きで投稿します。
- 「振り分け開始」から候補を選び、イベントへ採用できます。
- 採用時に内容の修正、担当者、期限を入力できます。
- 不要な候補は破棄できます。

## 出費の使い方

イベントスレッドのコントロールパネルから「💰 出費」を押します。

記録追加は賞金・景品対応担当またはイベント統括が可能です。

入力:

```text
カテゴリ: prize / gift / operation / other
方向: out または in
金額 Land: 18000
対象者: 任意。@ユーザー または ユーザーID
発生日とメモ:
2026-07-01
賞品配布
```

モーダル送信後、5分以内に画像付きメッセージを投稿すると証明画像として紐付けます。紐付け成功時は `CH_EXPENSE_LOG` に整形版を投稿します。5分以内に拾えなかった場合はイベントスレッドへ警告を出します。

閾値アラートの初期値:

- 単発: `50,000 Land`
- イベント合計: `300,000 Land`
- 月次合計: `1,000,000 Land`

## カレンダー・統計・振り返り

`/events` には「📅 カレンダー」「📊 統計」ボタンがあります。

カレンダーは JST・日曜始まりで、開催日時が設定されたイベント日を `[ ]` で表示します。「前月」「来月」で月移動できます。

統計は月別に以下を表示します。

- イベント数
- 出費カテゴリ別合計
- イベント別出費ランキング
- シリーズ開催回数
- 担当回数

振り返りリマインダーは、開催日時 +24時間にイベントスレッドへ投稿します。イベントがすでに `done` または `cancelled` の場合は何もしません。

## コマンド登録

```bash
pnpm run register
```

この実装ではギルドコマンドとして登録します。

## 開発起動

```bash
pnpm run dev
```

## 本番ビルド

```bash
pnpm run build
pnpm start
```

systemd 用の雛形は `ecosystem/event-bot.service` にあります。

## テスト

```bash
pnpm test
```
